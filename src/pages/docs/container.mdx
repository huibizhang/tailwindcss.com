## Usage

<!-- The `container` class sets the `max-width` of an element to match the `min-width` of the current breakpoint. This is useful if you'd prefer to design for a fixed set of screen sizes instead of trying to accommodate a fully fluid viewport. -->
`container` class 設定元素的 `max-width` 來匹配當前斷點的 `min-width` 。這對於想要依照固定螢幕尺寸而非動態。

<!-- Note that unlike containers you might have used in other frameworks, **Tailwind's container does not center itself automatically and does not have any built-in horizontal padding.** -->
請注意這可能跟你以往所使用的框架的容器 (container) 不一樣。 **Tailwind的容器 (container) 不會自動置中，也沒有內置的水瓶內間距。**

<!-- To center a container, use the `mx-auto` utility: -->
使用 `mx-auto` 功能來置中 container 。

```html
<div class="container **mx-auto**">
  <!-- ... -->
</div>
```

<!-- To add horizontal padding, use the `px-{size}` utilities: -->
使用 `px-{size}` 功能來增加水平內間距。

```html
<div class="container mx-auto **px-4**">
  <!-- ... -->
</div>
```

<!-- If you'd like to center your containers by default or include default horizontal padding, see the [customization options](#customizing) below. -->
假如你喜歡使用預設的方式來置中或者增加預設水平內間距，可以查看下面的[#自定義選項](#customizing)。

<!-- ## Responsive variants -->
## 響應式變化模式

<!-- The `container` class also includes responsive variants like `md:container` by default that allow you to make something behave like a container at only a certain breakpoint and up: -->
`container` class 也包含響應式變化模式。像是預設的 `md:container` 能讓你的某些東西在某個斷點或者某些斷點表現得像一個容器 (container) 。

```html
<!-- Full-width fluid until the `md` breakpoint, then lock to container -->
<div class="**md:container md:mx-auto**">
  <!-- ... -->
</div>
```

<!-- ## Customizing -->
## 客製化

<!-- ### Centering by default -->
### 預設置中

<!-- To center containers by default, set the `center` option to `true` in the `theme.container` section of your config file: -->
預設居中容器 (container) 。在你的設定檔裡 `theme.container` 中設定 `center` 選項為 `true` 。

```js
// tailwind.config.js
module.exports = {
  theme: {
    container: {
      center: true,
    },
  },
}
```

<!-- ### Horizontal padding -->
### 水瓶內間距

<!-- To add horizontal padding by default, specify the amount of padding you'd like using the `padding` option in the `theme.container` section of your config file: -->
預設加上水平內間距，請在你的設定檔裡 `theme.container` 中設定你所想要的數值在 `padding` 選項。

```js
// tailwind.config.js
module.exports = {
  theme: {
    container: {
      padding: '2rem',
    },
  },
}
```

<!-- If you want to specify a different padding amount for each breakpoint, use an object to provide a `default` value and any breakpoint-specific overrides: -->
假如你想在不同的斷點上設定不同的內間距，可以使用一個物件來在不同的斷點覆蓋 `default` 的值。

```js
// tailwind.config.js
module.exports = {
  theme: {
    container: {
      padding: {
        DEFAULT: '1rem',
        sm: '2rem',
        lg: '4rem',
        xl: '5rem',
        '2xl': '6rem',
      },
    },
  },
};
```

<!-- ### Disabling responsive variants -->
### 停用響應式變化模式

<!-- If you'd like to disable the responsive variants, you can do so using by setting `container` to an empty array in the `variants` section of your `tailwind.config.js` file: -->
倘若你想停用響應式變化模式，可以在你的 `tailwind.config.js` 文件裡的 `variants` 物件內的 `container` 設定為空陣列。


```diff-js
  // tailwind.config.js
  module.exports = {
    variants: {
      // ...
+     container: [],
    }
  }
```

<!-- ### Disabling entirely -->
### 全部停用

<!-- If you don't plan to use the `container` class in your project, you can disable it entirely by setting the `container` property to `false` in the `corePlugins` section of your config file: -->
如果你沒計劃在你的專案裡使用 `container` class ，你可以在你的設定檔裡的 `corePlugins` 物件內的 `container` 屬性設定為 `false` 來停用。

```diff-js
  // tailwind.config.js
  module.exports = {
    corePlugins: {
      // ...
+     container: false,
    }
  }
```
